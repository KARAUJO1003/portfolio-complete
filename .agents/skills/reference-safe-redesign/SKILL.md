# Reference Safe Redesign

## Regras

- Nao copie textos, imagens, logos, codigo, nomes de classes, assets ou identidade proprietaria da referencia.
- Extraia padroes transferiveis: layout, ritmo, densidade, proporcao, hierarquia, interacao e motion.
- Documente a referencia como influencia, nao como origem de componentes copiados.
- Mantenha diferencas claras: conteudo proprio, paleta adaptada, tokens proprios, componentes `ds` proprios.
- Use a referencia para calibrar qualidade visual, nao para clonar a marca.

## Checklist

Antes de editar:

- Identifique o que sera replicado: estrutura, spacing, tipo de card, tipo de navegacao, motion.
- Identifique o que nao sera replicado: conteudo, fotos, copy, icones proprietarios, marca.
- Prefira componentes locais em `components/ds`.
- Se usar blocks externos, adapte a composicao e os tokens ao projeto.

Ao finalizar:

- Verifique se o resultado parece da mesma familia de qualidade, mas nao identico.
- Remova textos e imagens da referencia.
- Mantenha o portfolio orientado aos objetivos do Kaesyo.
