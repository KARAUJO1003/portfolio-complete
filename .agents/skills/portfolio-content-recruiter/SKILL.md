# Portfolio Content Recruiter

## Objetivo

Tratar o portfolio como funil de recrutamento:

1. Provar capacidade rapidamente.
2. Mostrar poucos projetos fortes primeiro.
3. Transformar detalhes extensos em resumo com "ver mais".
4. Separar conteudo para landing, pagina detalhada e curriculo.

## Priorizacao Da Landing

- Projetos primeiro.
- Depois trajetoria/timeline.
- Depois stack em chips compactos.
- Depois sobre curto.
- GitHub e contato como suporte, nao como foco.

## Regra De Resumo

Se uma secao tiver muito conteudo:

- mostrar 1 destaque ou 3 itens no maximo;
- incluir botao/link `Ver mais`;
- mover o detalhe para pagina propria, modal, collapsible ou builder futuro.

## Criterios De Projeto

Cada projeto destacado deve responder:

- Qual problema resolveu?
- Qual foi seu papel?
- Que tecnologias importam para a vaga?
- O que demonstra sobre produto, UI, backend ou qualidade?

Evite:

- lista longa de tecnologias sem contexto;
- muitos projetos medianos no primeiro scroll;
- texto generico como "sistema moderno" sem resultado ou decisao tecnica.

## Curriculo

- Curriculo deve ser mais direto e ATS-friendly.
- Portfolio pode ser mais visual e seletivo.
- Conteudo compartilhado deve ter versoes curta e longa.
